А как быть, если мы пытаемся импортировать функцию, название которой уже использовано в нашем коде?
Для разрешения конфликтов имён можно использовать псевдонимы в импорте.

import { sum as mathSum } from './math.js';
console.log( mathSum(4) );

Ну и наконец, используя import *, можно импортировать все публичные элементы модуля:

import * as math from './math.js';
console.log( math.circleArea(5) );   
console.log( math.sum(1,2) );

компиляция CSS-модулей и использования import для подгрузки CSS-стилей в JS-файл.

/* components/submit-button.js */
import styles from './submit-button.css';

Чтобы в CSS-модулях иметь возможность использовать только одно имя класса,
но при этом не дублировать базовые стили кнопки для error и disabled,
используется ключевое слово composes. Оно помогает повторно использовать
уже объявленные в другом селекторе стили.

.button {
  /* basic styles for button */
}
.disabled {
  composes: button;
  /* anything that only applies to Disabled */
}
.error {
  composes: button;
  /* anything that only applies to Error */
}

Кроме того, CSS-модули позволяют нам с помощью composes ссылаться на стили из другого файла:
